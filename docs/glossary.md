# プロジェクト用語集 (Glossary)

## 概要

このドキュメントは、プロジェクト内で使用される用語の定義を管理します。

**更新日**: 2026-07-02

---

## ドメイン用語

### コミット履歴 / Commit History

**定義**: リポジトリに記録された全コミットの時系列一覧。

**説明**: 本アプリでは「新しい順」に表示する。各コミットはハッシュ・メッセージ・著者・日時・ブランチ名を持つ。

**関連用語**: CommitInfo, コミット

**英語表記**: Commit History / Commit Log

---

### 差分 / Diff

**定義**: 2つのコミット（またはファイルの2バージョン）の間の変更内容。

**説明**: 本アプリでは「あるコミットで何が変わったか」を表示するために使う。追加行（+）は緑、削除行（-）は赤で表示する。変更前後の文脈行（コンテキスト行）を 3 行含めて表示する。

**関連用語**: DiffFile, DiffHunk, DiffLine

**英語表記**: Diff / Patch

---

### コンテキスト行 / Context Line

**定義**: diff 表示において、変更された行の前後に含まれる「変更のない行」。

**説明**: 変更の前後 3 行をデフォルトで表示し、変更箇所の文脈を理解しやすくする。

**英語表記**: Context Line

---

### バイナリファイル / Binary File

**定義**: テキストとして解釈できないファイル（画像・実行ファイル・圧縮ファイル等）。

**説明**: diff 表示ができないため、「バイナリファイルのため差分を表示できません」と表示してスキップする。

**英語表記**: Binary File

---

### ブランチ / Branch

**定義**: コミット履歴の特定の地点を指す名前付きポインタ。

**説明**: 本アプリのコミット一覧では、そのコミットに紐づくブランチ名をバッジとして表示する。

**関連用語**: タグ, refs

**英語表記**: Branch

---

### タグ / Tag

**定義**: リリースバージョンなど特定のコミットに付ける不変のラベル。

**説明**: ブランチと同様にコミット行にバッジとして表示する。

**英語表記**: Tag

---

### refs

**定義**: ブランチ名・タグ名など、コミットを指す全ての名前付き参照の総称。

**説明**: `CommitInfo.refs` フィールドに格納し、コミット行にバッジとして表示する。

**英語表記**: References / refs

---

## 技術用語

### egui / eframe

**定義**: Rust 製の即時モード GUI ライブラリ（egui）と、それをウィンドウとして動かすフレームワーク（eframe）。

**本プロジェクトでの用途**: アプリの全 UI をこれで描画する。毎フレーム `update()` が呼ばれ、状態に基づいて UI を再描画する（即時モード）。

**バージョン**: 0.31.x

---

### git2

**定義**: `libgit2`（C ライブラリ）の Rust バインディング。Git の操作を Rust から行うためのクレート。

**本プロジェクトでの用途**: コミット一覧の取得・diff 生成のバックエンド。Git コマンドのプロセス起動なしに直接 Git オブジェクトを読む。

**バージョン**: 0.19.x

---

### libgit2

**定義**: Git の操作を提供する C 言語ライブラリ。Git 本体（コマンド）とは別の独立した実装。

**本プロジェクトでの用途**: `git2` クレートが内部的に使用する。静的リンクされるため、ユーザーの環境にインストール不要。

---

### serde

**定義**: Rust の標準的なシリアライズ/デシリアライズフレームワーク。

**本プロジェクトでの用途**: `AppConfig` 構造体を TOML ファイルに読み書きするために使用。`#[derive(Serialize, Deserialize)]` で自動実装。

---

### Cargo

**定義**: Rust のビルドシステム兼パッケージマネージャー。

**本プロジェクトでの用途**: ビルド (`cargo build`)、実行 (`cargo run`)、テスト (`cargo test`)、依存関係管理 (`Cargo.toml`) の全てに使用する。

---

### Cargo.lock

**定義**: 全ての依存クレートの正確なバージョンを記録したロックファイル。

**本プロジェクトでの用途**: **必ずコミットする**。これにより「動く状態」を再現性高く復元できる。

---

## 略語・頭字語

### MVP

**正式名称**: Minimum Viable Product

**意味**: 必要最小限の機能で動作するプロダクト。

**本プロジェクトでの使用**: 「コミット履歴一覧 + diff 確認」の 2 機能のみを実装した最初のバージョンを指す。

---

### PRD

**正式名称**: Product Requirements Document

**意味**: プロダクト要求定義書。何を作るか・誰のためか・どう成功を測るかを定義する。

**本プロジェクトでの使用**: `docs/product-requirements.md`

---

### diff

**正式名称**: difference（差分）

**意味**: 2つのファイルまたはコミット間の変更箇所。

**本プロジェクトでの使用**: コミット選択後に右ペインに表示する内容を指す。

---

## アーキテクチャ用語

### 即時モード GUI / Immediate Mode GUI

**定義**: 毎フレーム UI 全体を再描画するスタイルの GUI パターン。ウィジェットは「状態を持たず」、描画関数を呼ぶだけで UI を構成する。

**本プロジェクトでの適用**: egui が採用するパターン。`AppState` に全ての状態を持ち、毎フレームの `update()` で UI を再描画する。

**関連コンポーネント**: `eframe::App::update()`, `AppState`

---

### レイヤードアーキテクチャ / Layered Architecture

**定義**: システムを責務ごとの層（レイヤー）に分割し、依存の方向を一方向に保つアーキテクチャパターン。

**本プロジェクトでの適用**: `UI → AppState → Git ロジック → 永続化` の方向のみ依存を許可。逆方向・横断は禁止。

---

## データモデル用語

### AppState

**定義**: アプリ全体の状態を保持する中心的な構造体。

**主要フィールド**:
- `repo_path`: 現在開いているリポジトリパス
- `commits`: 読み込み済みのコミット一覧
- `selected_commit`: 選択中コミットのインデックス
- `diff_files`: 選択コミットの変更ファイル一覧
- `diff_hunks`: 選択ファイルの差分

**関連エンティティ**: `CommitInfo`, `DiffFile`, `DiffHunk`

---

### CommitInfo

**定義**: 1つのコミットに関する表示用情報をまとめた構造体。

**主要フィールド**:
- `short_id`: 短縮ハッシュ（7文字）
- `message`: コミットメッセージ（1行目）
- `author`: 著者名
- `time`: Unix タイムスタンプ
- `refs`: ブランチ名・タグ名のリスト

---

### DiffHunk

**定義**: diff の一塊（変更箇所のひとかたまり）。`@@ -x,y +a,b @@` ヘッダと差分行の集合。

**主要フィールド**:
- `header`: `@@ ... @@` 形式のヘッダ行
- `lines`: `DiffLine` のリスト

---

## エラー・例外

### GitError::NotARepository

**発生条件**: 指定されたパスが有効な Git リポジトリでない場合（`.git` ディレクトリが存在しない等）

**対処方法**: 「Git リポジトリが見つかりません: {path}」をユーザーに表示し、別のパスを指定してもらう

---

### GitError::LargeFile

**発生条件**: diff を生成しようとしたファイルのサイズが 1MB を超えた場合

**対処方法**: diff 生成をスキップし、「ファイルが大きすぎます（{size}MB）」を表示する

---

### GitError::BinaryFile

**発生条件**: diff を生成しようとしたファイルがバイナリファイルと判定された場合

**対処方法**: diff 生成をスキップし、「バイナリファイルのため差分を表示できません」を表示する
