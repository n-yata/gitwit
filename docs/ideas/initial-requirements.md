# 初期要求

## 解決したい課題

既存のGit GUIクライアント（SourceTree・GitKraken・GitHub Desktop等）に対する不満:

- **重い**: 起動・操作が遅い
- **複雑すぎる**: 使わない機能が多く、UIが煩雑
- **デザインが合わない**: VSCodeのライトモードっぽいシンプルで見やすいデザインが欲しい

## ターゲットユーザー

- 自分自身（まずは個人用）
- Windows ユーザー

## 主要機能

| 優先度 | 機能 |
|--------|------|
| MVP | コミット履歴の確認 |
| MVP | コミット間の差分(diff)の確認 |
| Post-MVP | コミット・ステージング |
| Post-MVP | プッシュ・プル |
| Post-MVP | ブランチの作成・切り替え・マージ |

## MVP の範囲

**「読むだけ」から始める。**

- 指定したリポジトリのコミット履歴を一覧表示
- コミットを選択して差分(diff)を確認
- 書き込み系操作（コミット・プッシュ等）はPost-MVP

## UIデザインリファレンス

**VSCode拡張の「Git Graph」をイメージ**

- 上部: コミット一覧 + ブランチのグラフ可視化（分岐・合流を線で表現）
- 下部: クリックで選択したコミットの変更ファイル一覧 + diff表示
- 白ベース・シンプル・フラットなデザイン

MVPではグラフ可視化は省略し、コミット一覧 + diff表示の2ペインから始める。

## 制約・前提

- **プラットフォーム**: Windows のみ
- **UIフレームワーク**: egui（純Rust製、軽量）
- **Gitバックエンド**: git2 クレート（libgit2バインディング）
- **言語**: Rust（開発者はRust未経験。Claude Codeと協力して実装）
- **デザイン方針**: VSCodeライトモード風（白ベース・シンプル・フラット）
- **配布形態**: 未定（自分用なので当面はビルドして使う）
