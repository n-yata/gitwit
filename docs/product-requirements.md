# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**Gitwit** - 軽量・シンプルな Rust 製 Git GUI クライアント

> ※ 名称は仮称。正式名称は後で決める。

### プロダクトコンセプト
- **軽さ**: 起動が速く、操作がサクサク動く。Electron 製ツールのような重さがない
- **シンプルさ**: 自分が使う機能だけを置く。使わないメニューやパネルは存在しない
- **見やすさ**: VSCode ライトモード風の白ベース・フラットなデザインで、目が疲れない

### プロダクトビジョン
「Git の状態を把握するためだけに重いツールを起動したくない」という不満から生まれた、自分専用の Git GUI クライアント。
コミット履歴の閲覧と差分確認という「読む」操作に特化した MVP から始め、使いながら必要な機能だけを足していく。
Rust + egui で作るため、ネイティブに近い軽さを実現する。

### 目的
- 既存 Git GUI クライアント（SourceTree・GitKraken 等）を起動せずに済む、自分専用の軽量代替ツールを作る
- VSCode の Git Graph 拡張に近いデザイン・使い勝手をスタンドアロンアプリで実現する
- Rust を Claude Code とともに学びながら実際に動くアプリを完成させる

---

## ターゲットユーザー

### プライマリーペルソナ: 自分自身（Windows 開発者）

- **環境**: Windows 11、Git インストール済み、主にターミナル + VSCode で開発
- **現在の課題**:
  - 差分を確認したいだけなのに、起動が遅い GUI ツールを待ちたくない
  - 使わない機能がUIに溢れていて、目的の操作にたどり着くのが手間
  - `git log --oneline` や `git diff` をターミナルで打つのは見づらい
- **期待する解決策**:
  - 起動したらすぐコミット一覧が見える
  - クリック一つで diff が表示される
  - 余計なものが画面に出ていない
- **典型的な利用シーン**:
  - 「さっきのコミットで何を変えたっけ？」と思ったときにサッと開く
  - PR を作る前にコミット履歴と差分を流し見する
  - 昨日の変更を朝に振り返る

## 成功指標(KPI)

自分用ツールのため、「ユーザー数」ではなく「自分にとっての使い勝手」で評価する。

### プライマリーKPI

- **起動時間**: 3 秒以内にコミット一覧が表示される（500 コミット規模のリポジトリで）
- **操作レスポンス**: コミット選択から diff 表示まで 200ms 以内
- **実用到達**: MVP 完成後 1 週間以内に、実際の開発作業で既存ツールの代わりに使い始められる

### セカンダリーKPI

- **クラッシュゼロ**: 通常操作でクラッシュしない
- **日常使用**: 週 5 回以上、実際の開発中に開くようになる（3 ヶ月後）

## 機能要件

### コア機能(MVP)

#### 機能1: リポジトリを開く

**ユーザーストーリー**:
開発者として、手元の Git リポジトリの履歴を見るために、フォルダを指定してアプリに読み込みたい

**受け入れ条件**:
- [ ] アプリ起動時にリポジトリのパスを指定できる（ファイルダイアログ or テキスト入力）
- [ ] 指定したパスが有効な Git リポジトリでない場合、エラーメッセージを表示する
- [ ] 前回開いたリポジトリのパスを記憶し、次回起動時に自動で開く

**優先度**: P0（必須）

---

#### 機能2: コミット履歴の一覧表示

**ユーザーストーリー**:
開発者として、リポジトリの変更経緯を把握するために、コミット一覧をわかりやすく見たい

**受け入れ条件**:
- [ ] コミット一覧を新しい順に表示する
- [ ] 各行に「コミットハッシュ（短縮）・コミットメッセージ・著者・日時」を表示する
- [ ] 500 件のコミットを 3 秒以内に表示する
- [ ] 一覧をスクロールして過去のコミットを遡れる
- [ ] ブランチ名・タグ名をコミット行にバッジとして表示する

**優先度**: P0（必須）

---

#### 機能3: 差分(diff)の表示

**ユーザーストーリー**:
開発者として、特定のコミットで何が変わったかを確認するために、選択したコミットの差分を見たい

**受け入れ条件**:
- [ ] コミット一覧でコミットをクリックすると、変更ファイル一覧を表示する
- [ ] ファイルを選択すると、そのファイルの diff を表示する（追加行は緑、削除行は赤）
- [ ] コミット選択から diff 表示まで 200ms 以内に完了する
- [ ] diff はコンテキスト行（変更前後 3 行）を含めて表示する
- [ ] バイナリファイルは「バイナリファイルのため差分を表示できません」と表示する

**優先度**: P0（必須）

### インターフェース仕様

**レイアウト（2ペイン構成）**:

```text
┌─────────────────────────────────────────────────────────┐
│ [リポジトリパス表示]                          [開く]     │  ← ツールバー
├──────────────────────────┬──────────────────────────────┤
│ コミット一覧              │ diff パネル                  │
│                          │                              │
│ ● abc123  feat: 新機能   │ [変更ファイル一覧]            │
│   田中太郎  2日前  [main]│ > src/main.rs                │
│                          │   src/lib.rs                 │
│ ○ def456  fix: バグ修正  ├──────────────────────────────┤
│   田中太郎  3日前        │ [diff 表示]                  │
│                          │ - old line                   │
│ ○ ghi789  docs: README   │ + new line                   │
│   田中太郎  4日前        │                              │
└──────────────────────────┴──────────────────────────────┘
```

- 左ペイン: コミット一覧（クリックで選択・右ペインに反映）
- 右上: 変更ファイル一覧（クリックで diff を切り替え）
- 右下: 選択ファイルの diff 表示
- 白ベース・フラットデザイン（VSCode ライトモード準拠）

### 将来的な機能(Post-MVP)

#### ステージング・コミット

作業ツリーの変更をステージして、コミットメッセージを書いてコミットできる。

**優先度**: P1（重要）

#### プッシュ・プル

リモートリポジトリとの同期（`git push` / `git pull` 相当）。

**優先度**: P1（重要）

#### ブランチ操作

ブランチの作成・切り替え・マージ・削除。

**優先度**: P1（重要）

#### ブランチグラフ可視化

コミット一覧にブランチの分岐・合流をグラフ（線）で視覚化する（Git Graph スタイル）。

**優先度**: P2（できれば）

## 非機能要件

### パフォーマンス
- 起動からコミット一覧表示まで: **3 秒以内**（500 コミット規模のリポジトリ）
- コミット選択から diff 表示まで: **200ms 以内**
- UI のフレームレート: 操作中に **60fps** を維持（egui のデフォルト動作）

### ユーザビリティ
- 初めて起動した状態から、コミット一覧を表示するまでの操作が **3 ステップ以内**
- エラー時は原因がわかるメッセージを表示する（「開けません」ではなく「Git リポジトリが見つかりません」など）

### 信頼性
- 通常操作（リポジトリを開く・コミット選択・diff 表示）でクラッシュしない
- 前回開いたリポジトリのパスを永続化し、再起動後も保持する

### セキュリティ
- 読み取り専用操作（MVP 範囲）のため、リポジトリへの書き込みは一切行わない
- ユーザーが指定したパス以外のファイルにはアクセスしない

### スケーラビリティ
- 1,000 コミット規模のリポジトリで動作する（それ以上は将来対応）
- 大きなファイルの diff（1MB 超）は警告を表示し、表示をスキップするオプションを提供する

## スコープ外

MVP および現時点で明示的にスコープ外とする項目:

- **macOS / Linux 対応**: Windows のみ。クロスプラットフォーム対応は将来検討
- **GitHub / GitLab 連携**: PR 作成・Issue 参照などの外部サービス連携
- **設定 UI**: カラーテーマ切り替え・フォントサイズ変更などのカスタマイズ画面
- **コンフリクト解消 UI**: マージコンフリクトの視覚的解消
- **クラウドセーブ / 設定同期**
- **キーボードショートカットのカスタマイズ**
