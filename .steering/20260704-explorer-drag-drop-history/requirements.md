# 要求内容

## 概要

起動中のアプリのウィンドウに Windows Explorer からファイル/フォルダをドラッグ&ドロップすると、そのパス(配下)のコミット履歴に絞り込んで表示できるようにする。

## 背景

現状、特定ファイル/フォルダ配下のコミット履歴に絞り込むには、Windows Explorer の右クリックメニュー(`scripts/register-context-menu.ps1` で登録した `Gitwit` シェルメニュー)からアプリを起動し直す必要がある。アプリを既に起動した状態で絞り込み対象を切り替えたい場合、都度右クリック→メニュー選択という手間が発生する。

起動中のアプリウィンドウに Explorer から直接ファイル/フォルダをドラッグ&ドロップできれば、右クリックの手間なく即座に絞り込み対象を切り替えられる。以下の既存資産をそのまま再利用できる見込み:

- `src/cli.rs` の `resolve_target()` — パス→(リポジトリルート, 相対パスの file_filter) 解決ロジック
- `src/app.rs` の `AppState.file_filter` + `needs_load` フラグ — 履歴の絞り込み反映の仕組み
- `eframe`(winit バックエンド)は追加依存なしで `ctx.input(|i| i.raw.dropped_files)` からOSのファイルドロップを取得可能

## 実装対象の機能

### 1. OSファイルドロップの検知

- `egui`/`eframe` の `dropped_files`(`ctx.input(|i| i.raw.dropped_files.clone())`)を `App::update` 内で読み取る
- ユーザーがアプリウィンドウにファイル/フォルダをドロップした瞬間に検知できるようになる

### 2. ドロップされたパスの解決と反映

- ドロップされたパスを `cli::resolve_target()` 相当のロジックでリポジトリルートと相対パス(file_filter)に解決する
- 解決結果を `AppState` に反映し、`needs_load = true` にして履歴一覧を再読み込みする
- 現在開いているリポジトリと異なるリポジトリのパスがドロップされた場合は、リポジトリごと切り替える(`repo_path` を差し替えて開き直す)
- ユーザーはドラッグ&ドロップだけで、任意のファイル/フォルダ配下の履歴、または別リポジトリの履歴に切り替えられるようになる

### 3. エラーハンドリング

- ドロップされたパスがGitリポジトリの配下でない場合、既存の `error_message` エラーモーダルを再利用してエラーメッセージを表示する
- 新規UIコンポーネントは追加しない

## 受け入れ条件

### OSファイルドロップの検知・反映

- [ ] 起動中のアプリのウィンドウにフォルダをドラッグ&ドロップすると、そのフォルダ配下のコミット履歴のみが一覧に表示される
- [ ] 起動中のアプリのウィンドウに単一ファイルをドラッグ&ドロップすると、そのファイルの変更を含むコミットのみが一覧に表示される
- [ ] トップバーの「履歴フィルタ: {path}」表示と「✕」クリアボタンは、ドロップ経由の絞り込みでも既存同様に機能する(クリアすると絞り込みが解除され全履歴が表示される)

### リポジトリ切り替え

- [ ] 現在開いているリポジトリと異なるリポジトリ配下のパスをドロップすると、そのリポジトリに切り替わり履歴が表示される

### 複数ドロップ・異常系

- [ ] 複数ファイル/フォルダを同時にドロップした場合、先頭の1件のみが対象として反映される
- [ ] Gitリポジトリの外にあるパスをドロップすると、既存のエラーモーダルでエラーメッセージが表示され、現在の表示状態(リポジトリ・履歴・フィルタ)は変化しない

## 成功指標

- 定性的: 右クリックメニューを使わずに、起動中のアプリだけで絞り込み対象・リポジトリを切り替えられる体験を提供する
- 定量的な目標は特に設定しない

## スコープ外

以下はこのフェーズでは実装しません:

- ドラッグ中(ホバー時、ドロップ前)の視覚的フィードバック/オーバーレイ演出
- 複数パスを同時に絞り込み対象とするマルチパスフィルタ(OR条件)
- 非Windows環境でのドロップ対応検証(本アプリはWindows専用)
