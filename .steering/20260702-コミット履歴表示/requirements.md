# 要求定義: コミット履歴表示

## 概要

Rust + egui + git2 を使った Git GUI クライアントの MVP 第一弾。
「リポジトリを開く」「コミット履歴一覧を表示する」の 2 機能を実装する。

## 実装スコープ

### 含む
- Cargo プロジェクトの初期化（`cargo new` 相当の構造）
- 依存クレートの追加（eframe, egui, git2, serde, toml, chrono）
- データモデル定義（CommitInfo, AppState, AppConfig）
- Git ロジックレイヤー（`src/git/`）: リポジトリオープン + コミット一覧取得
- 設定永続化（`src/config.rs`）: 前回のリポジトリパス保存・読み込み
- ツールバー UI（`src/ui/toolbar.rs`）: パス表示 + 「開く」ボタン（パス直接入力）
- コミット一覧 UI（`src/ui/commit_list.rs`）: コミット行表示・スクロール・選択
- エラー表示（無効なリポジトリパスの場合）
- ユニットテスト（日時フォーマット関数・CommitInfo 変換）

### 含まない（次フェーズ）
- diff 表示（別 feature として実装）
- ファイルダイアログ（rfd クレート等）: 初回はパスの直接テキスト入力のみ
- 仮想スクロール（初回は通常スクロール。1,000件超は後回し）

## 受け入れ条件

- [ ] `cargo build` がエラーなく通る
- [ ] アプリを起動するとウィンドウが開く
- [ ] テキストフィールドにリポジトリパスを入力して「開く」を押すとコミット一覧が表示される
- [ ] 各コミット行に「ショートハッシュ・メッセージ・著者・相対日時・ブランチ名バッジ」が表示される
- [ ] 無効なパスを入力するとエラーメッセージが表示される
- [ ] 前回開いたリポジトリパスが次回起動時に自動で読み込まれる
- [ ] `cargo test` がパスする
