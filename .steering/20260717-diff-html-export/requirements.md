# 要求内容

## 概要

コードレビューで共有できるように、差分パネルで表示している変更ファイルの一覧とその差分を、自己完結HTMLファイルとしてエクスポートできるようにする。

## 背景

現状のgit-clientはアプリ内でのみ差分を閲覧でき、差分内容を他者と共有する手段がない。コードレビューの際にアプリを持たない相手にも差分を見てもらえるよう、ブラウザだけで閲覧できる単一HTMLファイルとして出力したい。

## 実装対象の機能

### 1. 差分のHTMLエクスポート
- 差分パネル(diff_panel.rs)にエクスポート用のボタンを追加する
- 単一コミットの差分(親コミットとの比較)、および2コミット比較(Ctrl+Click選択)のどちらの表示中でもエクスポートできる
- ボタン押下時、rfdクレートによるネイティブの「名前を付けて保存」ダイアログを表示し、保存先パス・ファイル名をユーザーが指定する
- 生成されるHTMLは外部リソースに依存しない自己完結ファイル(CSS/JSはすべてインライン埋め込み)

### 2. ファイル一覧 + クリックで差分表示
- HTML内に変更ファイルの一覧を表示する。各行には以下を含む:
  - ファイルパス(リネームの場合は旧パスも表示)
  - 変更種別(追加 / 変更 / 削除 / リネーム)
  - 変更行数(+N / -N)
- 一覧の各ファイルをクリックすると、そのファイルの差分(side-by-side、既存アプリのdiff_panel.rsの表示に準ずる追加/削除/コンテキスト行の色分け)がHTML内に表示される
- バイナリファイルは一覧には表示するが、差分本体は表示せず「バイナリファイル」等の注記のみ表示する

## 受け入れ条件

### 差分のHTMLエクスポート
- [ ] 単一コミット表示中に差分パネルのエクスポートボタンを押すと、保存ダイアログが開く
- [ ] 2コミット比較表示中に差分パネルのエクスポートボタンを押すと、保存ダイアログが開く
- [ ] 保存後、生成されたHTMLファイルを外部ブラウザ(インターネット接続なし)で開いても崩れずに表示できる

### ファイル一覧 + クリックで差分表示
- [ ] HTMLを開くと変更ファイルの一覧(パス・変更種別・変更行数)が表示される
- [ ] 一覧のファイルをクリックすると、そのファイルの差分がside-by-sideで表示される
- [ ] リネームされたファイルは旧パスも一覧に表示される
- [ ] バイナリファイルは一覧に表示されるが、差分は表示されず注記のみ表示される

## 成功指標

- コードレビュー時に、アプリ未インストールの相手にもHTMLファイル1つの共有で差分内容が伝わる

## スコープ外

以下はこのフェーズでは実装しません:

- Markdown等HTML以外の出力形式
- メール送信・クラウド共有等のエクスポート後の配布機能
- 3コミット以上の範囲比較のエクスポート
- 生成HTML上での差分編集・コメント機能
