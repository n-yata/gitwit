# 要求内容

## 概要
現在開いているリポジトリのローカルブランチ一覧をツールバーに表示し、ドロップダウンから選択したブランチへワンクリックで切り替え（checkout）できるようにする。

## 背景
gitwit は現状「読むだけ」の MVP（コミット履歴一覧 + diff 確認）が完成しており、コミット行にはブランチ/タグのバッジ表示（`CommitInfo.refs`）が既に実装済み。しかし現在のブランチを確認したり、別ブランチへ切り替える手段がなく、ブランチをまたいだ調査のたびに外部ターミナルで `git checkout` する必要がある。CLAUDE.md のロードマップにも Post-MVP 機能として「ブランチの作成・切り替え・マージ」が挙げられており、その第一歩として「切り替え」のみを最小スコープで実装する。

## 実装対象の機能

### 1. 現在のブランチ表示
- ツールバー（画面上部）に現在チェックアウトされているブランチ名を常時表示する。
- HEAD が detached 状態の場合は「(detached HEAD)」等の専用表示にする。

### 2. ブランチ切り替え
- ツールバーのブランチ名表示をドロップダウンにし、クリックするとリポジトリ内のローカルブランチ一覧が表示される。
- 一覧から選択すると、選択したブランチへ checkout する（`git2::Repository` の checkout 相当）。
- 対象はローカルブランチのみ（リモートブランチ origin/xxx の一覧・チェックアウトは対象外）。
- 切り替え成功後、コミット一覧パネル・diff パネルを自動的にリフレッシュする（既存のコミット読み込みロジックを再利用）。

### 3. 未コミット変更がある場合の挙動
- 切り替え先で作業ツリーの変更が衝突する場合は checkout を実行せず、エラーメッセージを表示して処理を中断する（force checkout や stash 等の救済導線は今回のスコープ外）。

## 受け入れ条件

### 現在のブランチ表示
- [ ] リポジトリを開くと、ツールバーに現在の HEAD が指すブランチ名が表示される
- [ ] detached HEAD 状態のリポジトリを開くと「(detached HEAD)」等それとわかる表示になる

### ブランチ切り替え
- [ ] ブランチ名表示をクリックすると、ローカルブランチの一覧がドロップダウンで表示される
- [ ] 一覧から別のブランチを選択すると、実際に `HEAD` が該当ブランチに切り替わる（`git status` 相当で確認可能）
- [ ] 切り替え成功後、コミット一覧パネルが新しいブランチの履歴に更新される
- [ ] 切り替え成功後、diff パネルの選択状態はクリアされる
- [ ] リモートブランチ（origin/xxx）は一覧に表示されない

### 未コミット変更時のエラー処理
- [ ] 作業ツリーに切り替え先と衝突する未コミット変更がある状態で切り替えを試みると、checkout は実行されずエラーメッセージが表示される
- [ ] エラー表示後もアプリはクラッシュせず、元のブランチのまま操作を継続できる

## 成功指標
- 外部ターミナルを使わずに、gitwit の UI だけでブランチ切り替えが完結する

## スコープ外

以下はこのフェーズでは実装しません:

- リモートブランチ（origin/xxx）の一覧表示・チェックアウト
- ブランチの新規作成・削除・リネーム
- ブランチのマージ
- 未コミット変更がある場合の force checkout / stash 等の救済導線
- ブランチグラフの可視化（Post-MVP P2 で別途）
