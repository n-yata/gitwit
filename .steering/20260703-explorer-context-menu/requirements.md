# 要求内容

## 概要

Windows Explorer で右クリックしたファイル/フォルダから直接 Gitwit を起動し、対象の変更履歴を確認できるようにする。TortoiseGit の「ログを表示」相当の体験を、軽量なレジストリ登録 + CLI引数対応で実現する。

## 背景

現状 Gitwit はアプリを起動してからリポジトリパスを手入力する必要があり、「このファイルを最近誰がどう変更したか」を確認したい場面での動線が長い。TortoiseGit のような右クリック統合があれば、Explorer から即座に対象の履歴へアクセスできる。

ただし TortoiseGit 本体が使う COM シェル拡張(IContextMenu実装)は実装・保守コストが高いため、採用しない。Windows がネイティブにサポートする「静的コンテキストメニュー登録」(レジストリの `shell\<name>\command` キーでの外部プロセス起動)で必要十分な体験を実現する。

## 実装対象の機能

### 1. CLI引数によるターゲット指定起動

- `gitwit.exe "<path>"` の形式で起動すると、`<path>` が指すファイル/フォルダを含む Git リポジトリを自動検出して開く。
- `<path>` がファイルの場合: リポジトリルートを検出し、かつそのファイルを変更対象として履歴フィルタを適用する。
- `<path>` がフォルダの場合: リポジトリルートを検出して開くのみ(フィルタなし、通常の全コミット一覧)。
- 引数なしで起動した場合は既存動作(前回開いたリポジトリを自動ロード、なければ手入力待ち)を変更しない。

### 2. ファイル単位の履歴フィルタ

- 指定されたファイルパスが変更されているコミットのみをコミット一覧に表示する。
- フィルタ適用中であることをUI上に表示し(例: 「履歴フィルタ: src/app.rs」)、クリックでフィルタを解除して通常の全コミット一覧に戻せる。

### 3. Explorer 右クリックメニュー登録

- レジストリ (`HKEY_CURRENT_USER\Software\Classes\...`) に静的コンテキストメニュー項目を登録する PowerShell スクリプトを提供する。
  - `*\shell\Gitwit` : 任意のファイルを右クリックしたとき
  - `Directory\shell\Gitwit` : フォルダ自体を右クリックしたとき
  - `Directory\Background\shell\Gitwit` : フォルダの背景を右クリックしたとき
- 登録解除用のスクリプトも提供する。
- HKCU配下のみを使用し、管理者権限を必要としない。
- スクリプトはユーザーが手動実行する想定(自動実行はしない)。

## 受け入れ条件

### CLI引数によるターゲット指定起動
- [ ] `gitwit.exe "C:\repo\src\app.rs"` 実行時、`C:\repo` がリポジトリとして開かれる
- [ ] `gitwit.exe "C:\repo\src"` (フォルダ) 実行時、`C:\repo` がリポジトリとして開かれ、全コミットが表示される
- [ ] リポジトリでないパスや存在しないパスを渡した場合、エラーメッセージが表示され既存のエラーハンドリングパターンに従う
- [ ] 引数なし起動時の既存動作(前回リポジトリの自動ロード)に影響がない

### ファイル単位の履歴フィルタ
- [ ] ファイル指定時、そのファイルが変更されたコミットのみがコミット一覧に表示される
- [ ] フィルタ適用中である旨がUIに表示される
- [ ] フィルタを解除する操作ができ、解除後は全コミット一覧に戻る

### Explorer 右クリックメニュー登録
- [ ] 登録スクリプト実行後、任意のファイルを右クリックして「Gitwitで履歴を表示」的なメニューが表示される
- [ ] フォルダ自体・フォルダの背景の右クリックでも同様にメニューが表示される
- [ ] メニュークリックで Gitwit が起動し、対象の履歴が表示される
- [ ] 登録解除スクリプトを実行すると、メニュー項目が消える

## 成功指標

- Explorer からファイル右クリック→履歴表示までが2クリック以内で完結する
  - **既知の制約(Windows 11)**: Windows 11 の刷新された右クリックメニューでは、クラシックなレジストリ登録(`shell`キー)による項目は既定でトップレベルに表示されず、「その他のオプションを表示」を選んだ先のサブメニューに表示される。そのため実際の操作は「右クリック→その他のオプションを表示→Gitwitで履歴を表示」の3クリックになる。IExplorerCommand(COM)による新方式への対応、またはシステム全体のクラシックメニュー化(レジストリで`InprocServer32`既定値を空にする)という回避策はあるが、後者はGitwit以外の全コンテキストメニューの見た目・挙動に影響するためスコープ外とし、現状の3クリック挙動を受け入れる。

## スコープ外

- TortoiseGit 相当のCOMシェル拡張(アイコンオーバーレイ、動的サブメニュー等)の実装
- `git log --follow` 相当のリネーム追跡(ファイル名変更をまたいだ履歴追跡)
- HKLM(全ユーザー)へのレジストリ登録・インストーラー化
- 複数ファイル同時選択時の履歴表示
